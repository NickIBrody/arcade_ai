# Arcade AI

> Универсальный клиент языковых моделей для Android — тёмный, плавный, безопасный.

🇬🇧 [English version](README.md)

Arcade AI — одно мобильное приложение, которое общается с **любой** большой
языковой моделью. Вставляешь API-ключ, выбираешь провайдера, выбираешь модель —
и начинаешь чат. От мировых гигантов (OpenAI, Anthropic, Google) до российских
платформ (YandexGPT, GigaChat) и до своего собственного эндпоинта (Polza AI,
прокси или локальный сервер Ollama).

## Возможности

- **14 встроенных провайдеров** + полностью кастомный (любой эндпоинт,
  совместимый с форматом OpenAI или Anthropic).
- **Стриминг ответа** — текст появляется токен за токеном, как в веб-чатах.
- **Ход мыслей** — для думающих моделей (o-серия, DeepSeek R1, Claude с
  расширенным мышлением) раскрывающийся блок показывает, *как* модель рассуждала
  перед ответом.
- **Зрение (vision)** — можно прикрепить фото к сообщению там, где модель это поддерживает.
- **Сравнение двух моделей** — долгое нажатие на «отправить» задаёт один промпт
  двум моделям одновременно; оставляешь лучший ответ и продолжаешь с ним.
- **Два языка интерфейса** — английский / русский, выбор на экране приветствия,
  переключается позже в настройках.
- **Безопасность в основе** — API-ключи хранятся в аппаратном **Android Keystore**
  (AES-GCM); опциональная **биометрическая автоблокировка** при открытии.
- **Выверенный дизайн** — абсолютно чёрные поверхности, сдержанные фиолетовые
  акценты, анимированный фон-сияние и движение на каждом значимом переходе.

## Поддерживаемые провайдеры

| Мировые | Российские / региональные | Локальные и свои |
|---|---|---|
| OpenAI, Anthropic, Google Gemini, Groq, DeepSeek, xAI Grok, Mistral, Together AI, OpenRouter, Cohere, IBM Granite | YandexGPT (Алиса), GigaChat (Сбер) | Ollama, любой кастомный эндпоинт |

> Провайдеры, которым нужен особый обмен токенами OAuth/IAM (YandexGPT, GigaChat,
> watsonx), подключены с правильной формой запроса и помечены в интерфейсе для
> ручной настройки.

> **Локальные модели (Ollama)** подключаются к *запущенному серверу Ollama* по
> HTTP (`localhost:11434`) — приложение **не** загружает `.gguf` файлы напрямую.
> Полноценный локальный инференс GGUF (выбрать файл и запустить на телефоне через
> llama.cpp) — в планах, в этой сборке его нет.

## Почему такие языки?

Идея началась как «намешать языков, чтобы выглядело круто». Честный инженерный
ответ сформировал итоговый стек:

- **Dart + Flutter — всё приложение.** Flutter — это главная суперспособность
  Dart: единый код компилируется в нативный ARM, UI рисуется самим движком
  (Skia/Impeller) и выглядит пиксель-в-пиксель одинаково на любом устройстве, а
  hot reload занимает доли секунды. Для отполированного, насыщенного анимациями
  приложения это самый быстрый путь к результату.

- **Без Rust (пока) — и это и есть безопасный выбор.** Был соблазн добавить Rust
  ради «безопасности через своё шифрование». Но самописная криптография — это
  риск; **Android Keystore** хранит ключи в аппаратном защищённом модуле, откуда
  секрет не покидает чип, — это строго надёжнее, чем AES на уровне приложения.
  Приложение упирается в сеть (ждёт ответы API), поэтому горячей точки по CPU,
  которую стоило бы ускорять на Rust, попросту нет. Rust получит своё место в тот
  день, когда мы добавим **локальный инференс на устройстве** — вот это уже
  настоящая вычислительная задача, и именно под неё заложен задел.

Принцип: каждый язык должен приносить пользу, а не раздувать список модных слов.

## Удалённый SSH-терминал + OpenCode (экспериментально)

> **Работает.** Setup пишет конфиг-файл OpenCode
> (`~/.config/opencode/opencode.jsonc`) с твоим провайдером, base URL, ключом и
> моделью — OpenCode идёт на **твою** модель, а не на бесплатные встроенные.
> Фича ещё молодая (относись к терминалу как к раннему функционалу), но
> ключевой поток «твой провайдер в OpenCode» рабочий.

Отдельный экран **Терминал (BETA)** (открывается из боковой панели чата)
превращает приложение в тонкий клиент к агентной кодинг-сессии на **твоей
собственной машине** — никаких серверов с нашей стороны. Идея: принеси своё
железо (VPS, домашний ПК, малинку) по SSH и запусти
[OpenCode](https://github.com/sst/opencode) прямо с телефона.

**Поток:**
1. Выбираешь **«Мои данные»** (активный провайдер приложения + ключ + base URL +
   модель) или **«Free»** (бесплатные модели OpenCode, без ключа).
2. Вводишь машину: host / user / port / пароль (хранится зашифрованно в
   Keystore, опционально).
3. Выбираешь **режим запуска** — *с подтверждением* или *авто (без подтверждений)*.
4. В фоне идёт чистая пошаговая установка (подключение → определение пакетного
   менеджера → установка Node/git → установка OpenCode → запуск). Сырой вывод
   скрыт; при ошибке — сворачиваемый лог + повтор.
5. TUI OpenCode рисуется в настоящем эмуляторе терминала (`xterm`) поверх
   SSH-шелла, со спец-клавишами (Esc/Tab/Ctrl-C, Ctrl+буква, стрелки), вставкой
   из буфера и ручным режимом терминала для продвинутых.

Построено на `dartssh2` (SSH-клиент) + `xterm` (эмулятор терминала) — сам
протокол SSH и эмулятор это библиотеки; приложение — обвязка + провижн + UI.

**Известные шероховатости:**
- Твои провайдер/ключ/URL/модель пишутся в конфиг-файл OpenCode, и он идёт на
  твою модель. Выбери **«Мои данные»** для этого или **«Free»** для встроенных
  бесплатных моделей OpenCode.
- Режим *авто/bypass* использует best-effort переменную; точный флаг может
  отличаться в зависимости от версии OpenCode.
- Достижимость: одна сеть (LAN) работает сразу; машина за NAT требует Tailscale /
  публичный IP / проброс порта.
- Установка пакетов требует root или passwordless sudo; иначе терминал спросит
  sudo-пароль.

## Архитектура

```
lib/
├── core/        тема, глобальное состояние (ChangeNotifier)
├── models/      провайдер, сообщение, конфиг генерации
├── data/        каталог провайдеров · хранилище ключей в Keystore · настройки
├── services/    стриминговый LLM-клиент (форматы OpenAI + Anthropic)
├── ui/
│   ├── onboarding/   язык → сетка провайдеров → форма подключения
│   ├── chat/         экран чата · переключатель моделей
│   ├── settings/     настройки · экран биометрической блокировки
│   └── widgets/      фон-сияние · пузыри · блок размышлений
└── l10n/        строки EN / RU
```

## Сборка

```bash
flutter pub get
flutter build apk --release
# результат: build/app/outputs/flutter-apk/app-release.apk
```

Нужны Flutter (stable) и Android SDK. Минимум — Android 6.0 (API 23).

## Модель безопасности

| Угроза | Решение |
|---|---|
| Ключи в покое | Android Keystore через `flutter_secure_storage` (AES-GCM, аппаратный) |
| Повторный вход | Опциональная биометрия / код устройства (`local_auth`) |
| Сеть | HTTPS к провайдерам; cleartext только для `localhost` (Ollama) |
| Телеметрия | Нет — приложение обращается только к настроенному вами провайдеру |
